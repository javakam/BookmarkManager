# 书签工作台产品与 UE 设计

日期：2026-07-11  
状态：已按用户确认的方向定稿  
目标平台：Chrome、Edge 桌面版（Chromium / Manifest V3）

## 1. 背景

浏览器自带书签管理器适合少量书签，不适合长期积累后的检索、跨目录整理和安全清理。本产品直接管理浏览器原生书签，不建立第二套书签库。

设计以用户提供的真实导出文件 `bookmarks_2026_7_11.html` 为样本：

- 881 条书签，79 个文件夹，最大深度 4 层。
- 61.2% 的书签位于第 3～4 层。
- 69 组确定重复 URL，全部跨文件夹出现。
- 67 组重复集中在两组镜像目录，问题主要来自目录复制，而非零散重复收藏。
- 27 条书签为空标题，其中大部分位于书签栏。结合用户截图，这些空标题是为了只显示 favicon、节省书签栏空间的主动设计，不是数据质量问题。
- 35 条为本机、回环或局域网链接；另有 `file:` 等特殊协议。
- 465 个不同主机名，长尾明显，适合域名辅助检索，不适合按域名自动重组。

## 2. 产品定位

书签工作台是面向 500～5000 条浏览器书签的本地优先管理工具，解决四类问题：

1. 记得大概内容，但找不到书签。
2. 同一网址被保存到多个目录，不敢直接去重。
3. 目录层级长期累积，移动和合并风险高。
4. 不知道哪些公网链接已失效，又不能误删只在特定电脑或网络中可用的链接。

产品不做云端账号、团队协作、网页正文抓取、AI 摘要、语义向量检索或独立书签同步。

## 3. 核心原则

### 3.1 原生书签是唯一事实源

- 读取和修改 Chrome / Edge 原生书签树。
- 不复制一套可独立编辑的书签库。
- 本地存储只保存搜索索引、链接检测结果、设置和操作记录。
- 监听浏览器书签新增、修改、移动和删除事件，界面增量刷新。

### 3.2 日常使用与集中管理分离

- 新标签页负责搜索、打开、展开常用文件夹和复现书签栏。
- 独立管理页负责移动、改名、去重、相似项、链接检测、隔离和恢复。
- 破坏性操作不出现在新标签页。
- Chromium 的新标签页覆盖由 Manifest 静态声明，不能在运行时恢复成浏览器原生页面。因此发布两个构建变体：默认的“管理器版”不覆盖新标签页；“新标签版”在同一代码基础上增加新标签页覆盖。

### 3.3 不自动修改

- 不自动删除、移动、改名、合并目录或更新重定向地址。
- 所有变更先选择明确范围，再重新校验，再预览影响，最后由用户确认。
- 批量操作逐项记录结果，部分失败必须明确展示。

### 3.4 保守判断

- 空标题默认是合法的图标书签，不进入标题问题列表。
- 跨目录相同 URL 默认是“多处收藏”，不是可直接删除的垃圾。
- 一次网络失败不等于链接失效。
- 本机、局域网、IP 地址和特殊协议不进入失效删除策略。

## 4. 信息架构

### 4.1 插件图标弹窗

保持轻量，只提供：

- 搜索书签。
- 收藏当前页面。
- 打开完整管理页。
- 打开设置。

弹窗不承担批量整理。

### 4.2 新标签页

新标签页属于单独的“新标签版”构建，不提供伪装成运行时开关的设置。页面职责是“马上找到并打开”：

- 顶部固定全局模糊搜索。
- 可选展示书签栏镜像；默认关闭，避免与浏览器上方已经显示的原生书签栏重复。
- 用户开启镜像后，严格保留书签栏的真实顺序和 favicon-only 显示。
- 展示用户选择的常用文件夹。
- 展示上次访问的文件夹；只有通过本产品打开过的书签才可显示最近记录。
- 搜索结果可直接打开，或跳转管理页定位原目录。
- 默认不展示 881 条全量列表，不展示清理统计，不提供删除和目录合并。

### 4.3 独立管理页

管理页包含四个一级视图：

1. **概览**：书签总数、文件夹数、待确认任务、最近一次检测，不展示健康评分。
2. **浏览**：逐层目录导航、当前目录内容、全局搜索和批量选择。
3. **整理**：重复项、相似项、镜像目录和链接检测。
4. **操作记录**：每次变更的逐项结果和可恢复入口。

详情采用按需抽屉或对话框，不长期占用右侧栏。

### 4.4 设置

- 显示当前安装的是“管理器版”还是“新标签版”，并说明切换版本需要安装对应构建。
- 是否在新标签页显示书签栏镜像。
- 默认打开方式。
- 链接检测超时、并发数和忽略清单。
- 隔离区保留时间。
- 清除已完成的检测结果和历史记录；仍在隔离区中的书签恢复元数据不能被该操作删除。
- 权限说明与隐私说明。

## 5. 全局搜索

### 5.1 索引字段

搜索在本机建立轻量索引，覆盖：

- 标题。
- 域名。
- 完整 URL。
- 完整文件夹路径。
- 中文全拼和拼音首字母。

不读取或索引网页正文。

### 5.2 匹配与排序

从高到低考虑：

1. 标题、域名或 URL 完全匹配。
2. 标题或域名前缀匹配。
3. 标题近似匹配。
4. 文件夹路径匹配。
5. URL 其余部分匹配。

短查询限制模糊容错，避免一个字符产生大量无关结果。搜索结果显示匹配原因和高亮片段。

书签栏直接项目可以轻量加权，但不能掩盖更高质量的准确匹配。

### 5.3 图标书签

- 原始空标题永不自动补写。
- 书签栏和新标签页继续只显示 favicon。
- 搜索和管理页面使用域名作为只读辅助名称，并标明“仅图标显示”。
- 辅助名称不回写浏览器书签。

### 5.4 多处收藏

相同 URL 位于多个目录时：

- 搜索可折叠显示为“同一网址位于 N 个位置”。
- 展开后显示每条路径和书签栏位置。
- 搜索本身不合并数据，也不推荐删除。

### 5.5 打开行为

- 新标签页：单击或回车在当前标签打开；中键或组合键在后台标签打开。
- 管理页：在新标签打开，保留搜索词、选择项和滚动位置。
- “定位原位置”进入管理页最新路径并高亮书签；如果书签已被外部移动，先重新定位。
- 零结果只提供清空搜索和浏览文件夹，不擅自跳转网络搜索。

### 5.6 性能目标

- 5000 条书签下，索引就绪后的搜索反馈目标为 100ms 内。
- 书签树变化采用增量更新；索引异常时可全量重建。
- 大结果集采用虚拟列表，避免一次渲染全部节点。

## 6. 浏览与目录管理

### 6.1 导航模式

采用面包屑与逐层列式导航：

- 当前路径始终完整可见。
- 点击一层目录后，下一层在右侧展开。
- 层级过多或窗口较窄时折叠最早的列，但保留面包屑。
- 搜索可以限定当前目录或全库。

### 6.2 书签栏模式

- 显示与浏览器一致的顺序预览。
- favicon-only 项保持纯图标。
- 拖动前后均展示目标位置。
- 文件夹和书签可拖动排序，但落下前显示完整目标路径和影响数量。
- 外部同步导致顺序变化时，放下前重新校验；冲突则取消本次移动并刷新。

### 6.3 批量操作

只有勾选项目后才出现批量工具栏：

- 移动。
- 加入隔离区。
- 恢复。
- 在当前范围内检测链接。

不提供无差别“自动整理”。

### 6.4 特殊节点

- 浏览器根目录不可重命名、移动或删除。
- 企业管理或标记为不可修改的节点只读显示。
- 操作目标包含只读节点时，将其从可执行项中排除并说明原因。

## 7. 重复项设计

### 7.1 分类

1. **确定重复**：完整 URL 一致。
2. **规范化候选**：仅 `www`、默认端口、末尾斜杠或常见跟踪参数不同。
3. **宽松候选**：查询参数或片段不同，仅用于人工比较。
4. **镜像目录**：两个目录中存在大量相同 URL。

后 3 类都不能自动删除。

完全相同 URL 集的 3 个及以上目录合并为一个镜像目录组，不展开为 N×N 建议。其他高重合目录候选按每目录 top-K 与全局上限返回，并明确标记结果是否被截断。

### 7.2 分组界面

每一组并排显示：

- 标题或域名辅助名称。
- 完整 URL 差异。
- 原始目录路径。
- 是否位于书签栏以及顺序。
- 添加时间；来源时间不可靠时明确标注。
- 判定原因和置信等级。

用户逐项勾选“保留”“隔离”或“暂不处理”。默认不预选删除项。

### 7.3 跨目录重复

- 统一标记为“多处收藏”。
- 默认全部保留。
- 用户可以将多个位置合并到一个目标目录，或只隔离指定位置。
- 镜像目录提供目录级比较，但执行时仍拆成逐项操作和结果。

## 8. 相似项设计

相似分析仅使用现有元数据，不抓取网页正文：

- 标题分词、拼音和编辑距离。
- 同域名下路径相似度。
- URL 规范化后的结构相似度。
- 文件夹名称和目录内容重合度。

结果分为“高相似”“可能相关”，不使用“重复”措辞。每组展示相似依据，并排比较；不预选任何删除项。

标题相同但 URL 不同时，必须显示为一个完整的“标题冲突组”，不能展开成 N×N 对，也不能当作重复链接。近似结果按每条书签 top-K 和全局上限截断，并明确返回“结果已截断”，避免极端数据产生数万组无法审查的结果。

## 9. 链接检测

### 9.1 权限与启动

- 链接检测由用户在管理页主动启动。
- 首次启动时请求 `http://*/*` 和 `https://*/*` 可选主机权限；用户拒绝或只授权部分来源时，未授权项显示为“未授权”，不归入暂时失败。
- 未授权时搜索、浏览、去重和相似分析仍可使用。
- 完整 URL 不发送到开发者或第三方服务器。

### 9.2 默认排除

以下链接直接标记为“不检测”，永不进入失效删除候选：

- `localhost`、回环地址和本地域名。
- 私网、链路本地和保留地址。
- 所有直接使用 IPv4 或 IPv6 地址的 URL，包括公网 IP。
- 单标签局域网主机名。
- `file://`、`chrome://`、`edge://`、`ftp://` 和其他非 HTTP(S) 协议。
- 用户加入忽略清单的域名、目录或单条书签。
- URL 静态判断无法覆盖所有分离 DNS 或公网域名解析到内网的情况；用户可以增加域名后缀和网段规则，界面明确说明这是静态排除。

### 9.3 检测流程

- 按主机限流，避免对同一站点高并发请求。
- 请求使用 `credentials: omit`、`referrerPolicy: no-referrer`、禁用缓存，不发送用户 Cookie、Authorization 或 Referer。
- 优先使用 `HEAD`；遇到 405/501 或结果不明确时，使用带小范围 `Range` 的受限 `GET`，收到响应头后尽快取消正文读取，不保存正文。
- 逐跳处理重定向；每一跳重新执行协议、IP、主机和忽略规则。无法安全读取 `Location` 时标记为“跳转待确认”，不盲目跟随。
- 重定向到本机、局域网、IP 或特殊协议时立即标记为“不检测”，不继续请求。
- 每个请求有明确超时。
- V1 仅在管理页打开时运行；关闭页面即安全暂停。每一项或小批完成后写入 IndexedDB，下次进入时由用户显式继续。
- 扫描过程中书签变化时，已删除节点跳过，已修改 URL 重新排队。

### 9.4 状态

| 用户状态 | 典型原因 | 是否进入删除候选 |
| --- | --- | --- |
| 正常 | 最终响应为 2xx | 否 |
| 已跳转 | 3xx 后到达可访问页面 | 否，仅提供更新地址建议 |
| 需登录或被拦截 | 401、403、407、451 | 否 |
| 未授权 | 用户未授予该来源的主机权限或权限中途被撤销 | 否，先授权或加入忽略 |
| 暂时失败 | 超时、DNS/TLS、408、425、429、5xx、离线 | 否，允许稍后复检 |
| 疑似失效 | 第一次明确 404 或 410 | 否 |
| 确认失效 | 同一精确 URL 连续两次 404/410，间隔不少于 24 小时且期间无成功 | 是，仍需人工确认 |
| 不检测 | 本地、局域网、IP、特殊协议或忽略项 | 永不进入 |

其他未列出的 4xx、304、重定向循环和跨协议异常默认不可删除。URL 被修改后，旧 URL 的失败次数清零。首版不读取正文，因此不尝试判断软 404。

## 10. 删除与恢复策略

### 10.1 隔离优先

“删除”默认表示移动到浏览器原生书签中的专用隔离文件夹：`待删除（书签工作台）`。

- V1 只允许隔离叶子书签，文件夹及其后代不能批量隔离或永久清空。
- 隔离前显示原路径、原顺序和原因。
- 隔离文件夹会随浏览器原生同步，其他电脑仍能找到这些书签。
- 默认保留 30 天；到期只产生“可清空”提醒，不自动永久删除。
- 每条隔离记录保存原父节点、祖先路径、原 index、前后相邻节点锚点和节点指纹。
- 父目录与相邻锚点仍存在时准确恢复；否则先展示可解释的候选位置并由用户确认，不能静默放到根目录。

### 10.2 永久清空

- 只能在隔离区主动发起。
- 再次列出数量和不可完整恢复的风险。
- 执行前把标题、URL、祖先路径、父节点指纹、原 index 和相邻节点锚点写入独立恢复记录。
- 恢复只能尽力重建；如果原目录已不存在，则提示选择现有目录或重建路径。
- 历史记录可以清理，但只要对应书签仍在隔离区，其恢复记录就不能被清除。

### 10.3 进入隔离区的条件

- 用户主动勾选的重复或相似项。
- 用户主动勾选的确认失效公网链接。
- 用户手动选择的普通书签。

系统不自动勾选，不自动移动。

## 11. 统一安全操作链

所有写操作遵循：

1. 用户选择明确范围。
2. 生成不可变的预览草案。
3. 生成预览时读取节点并比较 ID、标题、URL、父目录和顺序。
4. 发现外部变化时，仅跳过冲突项，不扩大操作范围。
5. 用户确认后限制并发；每个节点在实际写入前再次校验指纹并按锚点重算目标顺序。
6. 每项记录成功、失败或冲突。
7. UI 展示真实结果，不把部分成功显示为全部成功。
8. 可逆操作写入操作记录并提供恢复。

### 11.1 并发与同步冲突

可能来自另一台电脑同步、浏览器自带管理器或其他扩展：

- 分析结果携带生成时间和节点指纹。
- 结果过期时提示刷新分析。
- 删除或移动必须在对应浏览器 API 调用前再次校验，避免确认后到写入之间的竞争窗口。
- 目标目录已删除时暂停该项并要求重新选择。
- 同一节点不能同时进入两个执行中的批量任务。

### 11.2 撤销限制

- 移动、改名和隔离只有在原父节点与顺序锚点仍存在时才可准确撤销。
- 永久删除恢复会创建新节点，浏览器 ID 会变化。
- 原目录或锚点不存在时必须展示候选位置并要求确认，不得静默恢复到根目录。
- 企业只读节点和浏览器特殊根节点不参与撤销写入。

## 12. 错误与空状态

必须覆盖：

- 没有任何书签。
- 搜索零结果。
- favicon 缺失或加载失败。
- 权限拒绝、权限撤销和主机权限仅部分授权。
- 浏览器书签 API 调用失败。
- 批量操作部分成功。
- 链接检测暂停、页面关闭、网络离线和恢复。
- 扫描期间节点被删除、移动或 URL 被修改。
- 目标目录不存在或不可写。
- 隔离区同名冲突。
- 操作记录损坏或版本不兼容。
- 存储空间不足。

每个错误都应给出当前影响、已完成数量、未完成数量和可执行的下一步；不只显示“发生错误”。

## 13. 视觉与交互规范

- 中文为默认界面语言。
- 工作型、安静、高信息密度，不使用营销式首页和大面积装饰卡片。
- 主体为白色与浅灰背景，状态色只用于明确反馈。
- 重复、相似和链接状态使用文字加颜色，不仅依赖颜色。
- 圆角不超过 8px。
- 工具栏图标使用 Lucide，并提供 tooltip 与无障碍名称。
- 字号不随视口宽度缩放；长标题单行截断并可查看完整内容。
- 固定工具栏、列表行和图标尺寸，避免动态内容造成布局跳动。
- 桌面最小目标宽度 900px；窄窗口折叠导航，不压缩操作文本到不可读。

### 13.1 固定设计系统

实现以 `.superpowers/brainstorm/.../content/product-architecture.html` 与 `search-flow.html` 的已确认结构为视觉依据，采用以下固定值：

- 页面工作区：`#f5f7f8`；主表面：`#ffffff`；表头/次级表面：`#f0f3f5`。
- 主文字：`#20262d`；次级文字：`#626d78`；弱文字：`#7a848f`。
- 边框：`#d2d8de`；弱分隔线：`#e0e4e8`。
- 主操作与焦点：`#216b5d`；选中背景：`#e7f0ed`。
- 警告：`#ad6b18`；危险：`#b24c4c`；成功：`#28745f`。
- 字体：`Microsoft YaHei`, `Segoe UI`, system-ui, sans-serif；字距固定为 `0`。
- 字号：辅助 11px、表格与控件 12px、正文 13px、区块标题 16px、页面标题 20px。
- 间距：4、8、12、16、20、24、32px；控件高度 32 或 36px；列表行 48px。
- 圆角：小控件 4px、输入框与面板 6px、模态框 8px；不使用胶囊形状。
- 顶栏高度 60px；展开侧栏 232px；折叠侧栏 56px；内容区域使用列表、表格和全宽分区，不使用卡片墙。
- Lucide 图标默认 16px、1.8px 描边；主工具按钮 18px；图标按钮固定 32×32px。
- 阴影只用于命令搜索浮层和模态框，普通列表与分区只用边框区分。

首屏允许出现的固定文案包括：`书签工作台`、`搜索书签、网址、域名或文件夹`、`概览`、`浏览`、`整理`、`操作记录`、`设置`。除错误、数据状态和用户内容外，不增加营销文案、功能口号或说明性大段文字。

## 14. 键盘与无障碍

- 搜索框可自动聚焦。
- 上下键移动搜索结果，回车打开，Esc 清空或关闭当前浮层。
- 列表支持 Shift 连选与 Ctrl 多选。
- 拖动操作必须有等价的“移动到”命令。
- 焦点样式清晰，所有交互可用键盘完成。
- 图标书签在管理页面使用域名作为无障碍名称，不修改原始标题。
- 状态信息通过文本和 `aria-live` 反馈。

## 15. 技术架构

### 15.1 技术栈

- WXT：Manifest V3 扩展工程与多入口构建。
- React + TypeScript：界面与状态组件。
- Lucide React：统一图标。
- Fuse.js：本地模糊匹配。
- pinyin-pro：中文拼音索引。
- Vitest + Testing Library：单元与组件测试。
- Playwright：扩展页面端到端与视觉验证。

不加载远程 JavaScript，不向外部服务器上传书签。

### 15.2 模块边界

- `bookmark-repository`：浏览器书签 API 的唯一适配层。
- `bookmark-index`：搜索索引与增量更新。
- `duplicate-analyzer`：确定重复、规范化候选和镜像目录。
- `similarity-analyzer`：标题、URL 和目录相似度。
- `analysis-worker`：在独立 Worker 中运行重复、镜像目录和相似分析，主线程只接收可取消的进度与结果消息。
- `link-checker`：排除规则、队列、状态机、限流和恢复。
- `operation-planner`：预览、节点指纹和冲突检查。
- `operation-executor`：逐项执行与真实结果。
- `operation-journal`：本地操作记录与恢复信息。
- `ui-state`：只保存页面临时状态，不直接调用浏览器 API。

UI 只能通过这些边界调用业务能力，不能在组件中直接散落写操作。

### 15.3 存储

- 浏览器书签树：唯一主数据。
- `storage.local`：设置、检测结果摘要和索引版本。
- IndexedDB：从第一版起保存操作记录、恢复快照和链接扫描队列，不在运行中切换存储方案。
- `storage.sync` 只保存小型偏好设置，不保存完整 URL 清单。

### 15.4 权限

必需权限：

- `bookmarks`
- `storage`
- `activeTab`（弹窗中的“收藏当前页面”）
- `favicon`（使用浏览器提供的 favicon 缓存接口；缺失时使用中性占位）

链接检测时再申请：

- `http://*/*`
- `https://*/*`

这两项声明在 Manifest 的 `optional_host_permissions`，不放入安装时必需权限。打开扩展全页不为此单独申请 `tabs` 权限。首版不申请 `history`、`unlimitedStorage` 或内容脚本权限。

### 16.1 领域单元测试

- URL 规范化、查询参数和片段边界。

- 安装管理器版扩展并读取测试书签树。

- 移动、隔离和恢复。
- 外部修改发生在预览与执行之间时阻止冲突项。

- Chrome 与 Edge 构建产物检查。

### 16.4 视觉验证

- 1366×768、1440×900、1920×1080。
- 125% 与 150% Windows 缩放下文字不截断。
- 长标题、缺失 favicon、深层路径和大量结果不重叠。
- 键盘焦点、状态文本和对话框层级清晰。

## 17. 分阶段交付

### 17.1 V1a：先可试用的管理器版

- Chrome / Edge Manifest V3 管理器版构建，不覆盖新标签页。
- 插件弹窗、独立管理页和设置页。
- 原生书签树读取、逐层浏览、搜索、移动和叶子书签隔离恢复。
- 本地模糊搜索、拼音辅助和图标书签识别。
- 确定重复、规范化候选和基础相似项的只读结果。
- 操作预览、冲突检测、逐项结果和本地操作记录。

### 17.2 V1b：整理与检测

- 重复组和相似组的人工确认与隔离。
- 用户主动链接检测、逐跳排除规则和多状态结果。
- IndexedDB 扫描队列、暂停和显式继续。
- 失败复检计数和确认失效候选。

### 17.3 后续构建

- 新标签版构建（Manifest 增加 `chrome_url_overrides.newtab`）。
- 书签栏镜像和空间书架视图。
- 文件夹级合并预览。
- 永久清空隔离区、完整递归快照和更强的恢复向导。

所有阶段都不包含：

- 云端同步或账号。
- 网页正文索引、截图、AI 摘要或语义搜索。
- 自动定时检测。
- 自动整理、自动删除和自动更新重定向地址。
- 团队共享和移动端。

## 18. 验收标准

- 固定 5000 条 fixture 树读取后，以 `id/parentId/index/title/url` 的树哈希验证节点和顺序一致。
- 索引预热后执行 20 个固定查询，p95 反馈时间小于 100ms，并解释匹配字段。
- 图标书签保持空标题和顺序，不进入问题清单。
- 69 组确定重复可以按组展示原路径，不自动勾选任何删除项。
- 所有可由 URL 静态识别的本机、私网、单标签主机、所有 IP 和特殊协议标为“不检测”；自定义内网域名通过忽略规则覆盖。
- 任何重定向到排除地址的请求不继续访问，并标记为“不检测”或“跳转待确认”。
- 一次失败、认证限制、超时或 5xx 不进入删除候选。
- 所有移动、隔离和批量处理在执行前展示影响范围。
- 预览后发生外部修改时，冲突项不执行。
- 部分失败时展示逐项结果，成功项与失败项数量准确。
- 隔离叶子书签在父节点和锚点存在时可恢复到原路径和原顺序；路径不存在时要求用户选择，不静默放错位置。
- 页面在 1366×768、1440×900、1920×1080 及 125%/150% Windows 缩放下无文本重叠或关键操作遮挡。
